# AI Generation Contract

## Objective

Define the boundary between deterministic system components and AI-generated content.

This document establishes:

- What data enters AI
- What AI is allowed to generate
- What AI is NOT allowed to generate
- Validation requirements
- Failure handling
- Hallucination prevention

---

## Core Principle

AI may rewrite.
AI may summarize.
AI may improve wording.
AI may NOT invent facts.

Everything generated by AI must be traceable to evidence already present in the system.

---

## Inputs To AI

Source:

- Prompt Package
- Resume Draft

Schema:

```json
{
  "job_title": "",
  "match_score": 0,
  "strongest_skills": [],
  "matched_skills": [],
  "critical_gaps": [],
  "resume_recommendations": [],
  "career_recommendations": [],
  "resume_draft": {}
}
```

---

## Allowed AI Tasks

### Professional Summary

Transform structured candidate information into professional summary text.

Allowed:
- Rewriting
- Reordering
- Simplifying
- Improving readability

Not Allowed:
- New certifications
- New degrees
- New job titles
- New years of experience
- New skills

Token Limit: `max_tokens = 120`

---

### Experience Bullets

Transform evidence into achievement-oriented bullet points.

Allowed:
- Rewrite evidence
- Improve clarity
- Improve grammar
- Improve readability

Not Allowed:
- Invent metrics
- Invent percentages
- Invent KPIs
- Invent technologies
- Invent responsibilities
- Invent achievements

Token Limit: `max_tokens = 300`

---

### Skills Presentation

Allowed:
- Grouping
- Categorization
- Reordering

Not Allowed:
- Add skills not present in profile
- Rename skills into unsupported alternatives

Token Limit: `max_tokens = 80`

---

## AI Output Schema

Single source of truth. All validators and hallucination checkers reference this schema.

```json
{
  "summary": "",
  "experience_bullets": [
    {
      "text": "",
      "source_evidence_id": ""
    }
  ],
  "skills": {
    "technical": [],
    "soft": [],
    "domain": []
  }
}
```

### Field Definitions

| Field | Source | Notes |
|---|---|---|
| `summary` | Profile + matched skills | Rewritten, not invented |
| `experience_bullets[].text` | Evidence database | Rewritten from linked evidence only |
| `experience_bullets[].source_evidence_id` | Evidence database | Must resolve to existing evidence record |
| `skills.technical` | `data/reference/skills.json` | AI may group, not add |
| `skills.soft` | `data/reference/soft_skills.json` | AI may group, not add |
| `skills.domain` | Reserved for V2 | Industry taxonomy not yet defined. Pass empty list in V1. |

Token Limit (Full Resume): `max_tokens = 700`

---

## Validation Layer

Every AI response must pass validation before reaching the Hallucination Checker.

Checks:

1. Required keys exist (`summary`, `experience_bullets`, `skills`)
2. Correct data types (`summary` is string, `experience_bullets` is list, `skills` is object)
3. No empty response
4. No unsupported fields
5. Every `source_evidence_id` exists in the evidence database
6. No bullet has empty `text`
7. All skills originate from validated profile reference data
8. `summary` is not empty

If any check fails, trigger the Failure Handling flow. Do not proceed to the Hallucination Checker.

---

## Hallucination Detection

### Rule

Every claim must be traceable to evidence already in the system.

### V1 Mechanism: Keyword Overlap

A generated bullet passes if at least one meaningful content keyword from the bullet text appears in the linked evidence text.

Exclusions:
- Stopwords are excluded from keyword matching: `a, the, and, with, of, in, to, for, is, was, on, at, by`
- Minimum keyword length: 4 characters

### Examples

**Pass:**

```
Generated: "Led stakeholder communication across multiple teams."
Evidence:  "Coordinated communication with project stakeholders."
Result:    Pass  →  "communication", "stakeholder" overlap confirmed
```

**Fail:**

```
Generated: "Managed a team of 50 employees."
Evidence:  No mention of management or team leadership.
Result:    Fail  →  no keyword overlap found
```

### Failure Handling

- If a bullet fails: remove that bullet only. Remaining valid bullets continue.
- If all bullets fail: trigger fallback to deterministic resume draft.

### V1 Limitations

- No embeddings
- No semantic similarity
- No LLM-based judgment

### V2 Possibilities

- Embedding similarity
- Semantic matching
- Evidence ranking

---

## Failure Handling

### Retry Conditions

Retry once on:

- Invalid JSON
- Missing required keys
- Empty response

Retry delay: 2 seconds
Maximum retries: 1

### No-Retry Conditions

Do not retry on:

- Timeout
- API authentication failure
- Rate limit error

### Fallback

If retry fails, use deterministic resume draft. Do not surface a broken AI response to the user.

---

## Pipeline Architecture

```
Prompt Builder
      ↓
Resume Draft Builder
      ↓
AI Generator
      ↓
Validator          ← structural checks (schema, types, keys)
      ↓
Hallucination Checker  ← semantic checks (keyword overlap, evidence tracing)
      ↓
Resume Output
```

Validator runs before Hallucination Checker.
A structurally invalid response never reaches evidence tracing.

---

## Search Grounding Boundary

Search Grounding applies only to market-context outputs:

- UAE job trends
- salary insights
- industry demand
- hiring trend summaries

Search Grounding is never applied to candidate-data generation:

- professional summaries
- experience bullets
- skills
- credentials
- achievements
- resume claims

---

## Success Criteria

- No invented credentials
- No invented experience
- No invented metrics
- No invented technologies
- All generated content traceable to evidence
- Every bullet linked to a valid evidence ID
- Validator rejects malformed responses
- Hallucination checker removes unsupported claims
- Deterministic fallback available at every failure point
