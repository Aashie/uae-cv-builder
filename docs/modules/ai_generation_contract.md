# AI Generation Contract

## Objective

Define the boundary between deterministic system components and AI-generated content.

This document establishes:

- What data enters AI
- What AI is allowed to generate
- What AI is NOT allowed to generate
- Validation requirements
- Failure handling
- Hallucination prevention

---

# Core Principle

AI may rewrite.

AI may summarize.

AI may improve wording.

AI may NOT invent facts.

Everything generated by AI must be traceable to evidence already present in the system.

---

# Allowed AI Tasks

## Professional Summary

Transform structured candidate information into professional summary text.

Allowed:

- Rewriting
- Reordering
- Simplifying
- Improving readability

Not Allowed:

- New certifications
- New degrees
- New job titles
- New years of experience

---

## Experience Bullets

Transform evidence into achievement-oriented bullet points.

Allowed:

- Rewrite evidence
- Improve clarity
- Improve grammar

Not Allowed:

- Invent metrics
- Invent percentages
- Invent KPIs
- Invent technologies

---

## Skills Presentation

Allowed:

- Grouping
- Categorization

Not Allowed:

- Add skills not present in profile

# Validation Layer

Every AI response must pass validation.

Checks:

1. Required keys exist
2. Data types correct
3. No empty response
4. No unsupported fields

---

# Hallucination Detection

Rule:

Every claim must be traceable.

Validation:

Generated Output
↓
Evidence Database

If claim cannot be linked to evidence:

Flag as Hallucination

Examples:

Claim:
"Managed 50 staff"

Evidence:
No supporting evidence

Result:
Reject

---

# Failure Handling

If AI returns:

- Invalid JSON
- Missing fields
- Empty output

System:

Retry once

If retry fails:

Fallback to deterministic resume draft

# V1 Hallucination Detection Mechanism

V1 uses keyword-overlap checking.

A generated claim passes if at least one meaningful content keyword from the generated text appears in the linked evidence text.

V1 does not use embeddings, semantic similarity, or LLM judgment.

V2 may add semantic similarity with embeddings.

---

# Traceable Bullet Schema

Experience Bullets must include source evidence IDs.

{
  "bullets": [
    {
      "text": "",
      "source_evidence_id": ""
    }
  ]
}

Validation:

- source_evidence_id must exist.
- text must not be empty.
- bullet must be checked against the linked evidence text.

---

# Skills Output Schema

{
  "skills": {
    "technical": [],
    "soft": [],
    "domain": []
  }
}

Rules:

- Skills must come from existing profile skills.
- AI may group skills.
- AI may not add new skills.

---

# Retry Rules

Retry conditions:

- Invalid JSON
- Missing required keys
- Empty response

No-retry conditions:

- Timeout
- API authentication failure
- Rate limit error

Retry delay:

2 seconds

Maximum retries:

1

Fallback:

If retry fails, use deterministic resume draft.

# Success Criteria

- No invented credentials
- No invented experience
- No invented metrics
- All generated content traceable to evidence
- Deterministic fallback available
