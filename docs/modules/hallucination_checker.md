# Hallucination Checker

## Objective

Detect unsupported AI-generated experience bullets by verifying that each bullet can be traced to its linked evidence record.

The Hallucination Checker performs evidence validation only.

It does NOT validate schema structure.
That responsibility belongs to the AI Response Validator.

---

## Deliverable

Files:

- engine/hallucination_checker.py
- tests/test_hallucination_checker.py

---

## Inputs

### AI Output

```json
{
  "experience_bullets": [
    {
      "text": "",
      "source_evidence_id": ""
    }
  ]
}
```

### Evidence Database

Evidence objects generated by the Evidence Extractor.

Example:

```json
[
  {
    "id": "evidence_001",
    "text": "Coordinated communication with project stakeholders."
  }
]
```

---

## Scope

V1 checks:

- experience_bullets only

V1 does NOT check:

- summary
- skills

Summary hallucination detection is deferred to V2.

---

## Source Of Truth

The Hallucination Checker never reads directly from:

- master_profile.json

The checker must use:

```text
source_evidence_id
        ↓
Evidence Database
        ↓
Evidence.text
```

The Evidence Database is the only approved source of truth.

---

## Validation Mechanism

### V1 Rule

A bullet passes if at least one meaningful keyword from the generated bullet appears in the linked evidence text.

### Meaningful Keyword Definition

A keyword must:

- not be a stopword
- contain at least 4 characters

### Stopwords

```text
a
an
the
and
with
of
in
to
for
is
was
on
at
by
```

---

## Examples

### Pass

Generated Bullet:

```text
Led stakeholder communication across multiple teams.
```

Evidence:

```text
Coordinated communication with project stakeholders.
```

Result:

```text
PASS
```

Reason:

```text
communication
stakeholder
```

overlap exists.

---

### Fail

Generated Bullet:

```text
Managed a team of 50 employees.
```

Evidence:

```text
Handled client communications.
```

Result:

```text
FAIL
```

Reason:

No meaningful keyword overlap.

---

## Failure Handling

If a bullet fails:

- remove that bullet from passed results
- add it to failed results

If all bullets fail:

- hallucination_count equals total bullets
- downstream system may trigger deterministic fallback

The Hallucination Checker itself does not trigger fallback.

It only reports results.

---

## Output Format

```json
{
  "passed_bullets": [],
  "failed_bullets": [
    {
      "bullet": "",
      "reason": ""
    }
  ],
  "hallucination_count": 0
}
```

---

## Output Rules

Passed bullets:

- retain original structure

Failed bullets:

```json
{
  "bullet": "Managed a team of 50 employees.",
  "reason": "No keyword overlap with linked evidence."
}
```

---

## Success Criteria

- Uses linked evidence only
- Never accesses master_profile.json
- Ignores stopwords
- Enforces minimum keyword length
- Correctly separates passed and failed bullets
- Reports hallucination count
- Produces deterministic output

---

## Tests

- Valid overlap passes
- No overlap fails
- Missing evidence ID fails
- Stopwords ignored
- Minimum length enforced
- Multiple bullets mixed pass/fail
- All bullets fail
- Empty bullet list
- Empty evidence database
- Correct hallucination count
